About Me

My Story

During the last semester of my B.S. studies, I had a few "elective" courses remaining, and after a quick coin toss, I was enrolled in "Introduction to Programming." A few weeks into the course, I found myself thinking: "If I had known this was so much fun, I would have chosen programming as my major!" Alas, it was much too late for that, so I gobbled up as much information as I could from that course. After I graduated, I tried to pursue learning JavaScript on my own, but it proved to be a daunting task (to say the least!). Luckily, a few months later, I stumbled upon Free Code Camp and dove in —— head first, of course!. Now, I am armed with enough knowledge to build a pretty nifty website, as you can see below, in my portfolio.

Chrome Extension

Having built a weather API project at Free Code Camp, and an AngularJS project via Coursera, I wanted to learn how to integrate API calls into an AngularJS application. Luckily, I stumbled upon a Chrome Extension tutorial at ng-newsletter that did just that.


From this project, I not only learned how to make API calls with AngularJS, but I also learned a few "ins-n-outs" of developing a Chrome extension (e.g., the necessity of the manifest.json, the icon_128.png, and vendor permissions).


Much to my surprise, I was able to use my new skills (obtained from the tutorial) to expand on the project and customize my extension to include the expected weather conditions and the daily low temperatures.


Certainly, I feel more comfortable with making API calls in AngularJS! Additionally, I no longer have to stop what I'm doing when my husband asks, "What's the weather for tomorrow?" Now, I can simply open a new browser tab and give him the lowdown.  

Blog: The Chromebook Coder

Admittedly, I used a pre-made theme to scaffold out this project. My Ruby skills are beginner level, and I wanted to make sure my blog looked (and worked) like it was designed by a true professional. That being said, the process (which is somewhat detailed in a blog post) was not easy.


Despite the use of a pre-made theme, I have really learned a lot about the tools used to construct it. First, I had to figure out how to integrate all of the necessary tools into Cloud9 (my favorite IDE). Then, I had to learn how to write my posts using kramdown, a Ruby-flavored superset of Markdown syntax. I also had to learn how all of the pieces fit and work together (i.e., layouts, includes, data, etc.), in order to customize my blog.


Additionally, I learned how to use Disqus to add comments/discussions to my blog posts; how to make Twitter Cards and Open Graph cards for Facebook; and how to use Google Analytics and implement Adsense. Undoubtedly, it has been a very rewarding experience.

Lucky Leprechauns: A Matching Game

This project is based on a project I completed for a course at The Hong Kong University of Science and Technology (via Coursera). I chose to revisit the project to brush up on my "vanilla" JavaScript skills, and to revisit exploring the DOM structure.

Further, the initial project was constructed without Bootstrap, but I wanted to add some features unique to the framework (e.g. the jumbotron class header and a grid-based layout). I must say, overall, rehashing the project helped me accomplish my goals:

  • I was able to practice using JavaScript without relying on a specific library
  • I was able to revisit the DOM structure
  • And, I developed an added appreciation for Bootstrap's unique capabilities
